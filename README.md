# F28335 ePWM + thermistor Protection Example

## Overview

This project runs on the **TI TMS320F28335** and generates **two complementary PWM outputs** using **ePWM1** and **ePWM2**. It also monitors an analog signal on **ADCINA0** and **shuts down the PWM outputs** when the measured ADC value falls below a defined threshold.

After shutdown, the code waits for a restart delay and then checks the ADC again. The PWM outputs are only re-enabled if the sensed value has recovered above the threshold.

This is useful for protection-oriented applications such as:

- undervoltage lockout
- temperature-based shutdown
- analog fault monitoring
- safe restart after a fault clears
- testing phase-shifted PWM with ADC-based interlock

---

## Main features

- F28335 system initialization
- Two ePWM modules configured
- Complementary PWM generation with deadband
- ePWM2 phase shift relative to ePWM1
- CPU Timer0 used as a periodic control loop trigger
- ADCINA0 sampled in software
- PWM shutdown when ADC input goes below threshold
- Automatic restart only after delay **and** threshold recovery
- Debug/watch variables exposed for CCS

---

## Functional behavior

The program does the following:

1. Initializes system control, flash, PIE, timer, ADC, and ePWM
2. Starts a periodic loop using **CPU Timer0**
3. Reads ADC channel **A0**
4. Converts ADC to voltage for debugging
5. Ignores ADC trips during startup for a short period
6. If ADC goes below threshold:
   - forces both PWM modules low
   - latches a fault state
7. While fault is active:
   - waits for the restart delay
   - checks ADC again
   - restarts PWM only if ADC is back above threshold

---

## PWM structure

### ePWM1
- Base PWM reference
- Complementary outputs enabled through deadband

### ePWM2
- Same PWM structure as ePWM1
- Phase shifted by `PWM_DELAY_NS`
- Also configured as complementary PWM

This means ePWM2 is synchronized to ePWM1 but delayed in phase.

## User-configurable settings

These are the key parameters at the top of the code:

```c
#define DSPCLK                   (150000000UL)
#define FS_SWITCH                (70000UL)
#define FS_MOD                   (5000UL)
#define EPWM_DB                  (35U)

#define PWM_DELAY_NS             (150U)
#define ADC0_RAW_SHUTDOWN_THRESHOLD   (3186U)

#define PWM_RESTART_DELAY_COUNT  (FS_MOD * 10UL)
#define ADC_STARTUP_IGNORE_COUNT (1000U)
```

### Parameter meaning

#### `DSPCLK`
DSP clock frequency in Hz.

- Set here to **150 MHz**

#### `FS_SWITCH`
PWM switching frequency in Hz.

- Set here to **70 kHz**

#### `FS_MOD`
Main software loop frequency in Hz, generated by CPU Timer0.

- Set here to **5 kHz**

#### `EPWM_DB`
Deadband count applied to complementary PWM outputs.

#### `PWM_DELAY_NS`
Requested phase delay for ePWM2, in nanoseconds.

- Set here to **150 ns**

#### `ADC0_RAW_SHUTDOWN_THRESHOLD`
ADC threshold used to decide when PWM must shut down.

#### `PWM_RESTART_DELAY_COUNT`
Number of timer-loop cycles to wait before attempting restart.

#### `ADC_STARTUP_IGNORE_COUNT`
Number of initial control-loop cycles during which ADC trips are ignored.

## Timing calculations

### PWM period

The code defines:

```c
#define PERIOD (DSPCLK / (2UL * FS_SWITCH))
```

For the current values:

- `DSPCLK = 150 MHz`
- `FS_SWITCH = 70 kHz`

So:

```text
PERIOD = 150,000,000 / (2 × 70,000)
       ≈ 1071 counts
```

This is used for **up-down counting mode**, which is why the factor of 2 appears.

---

### ePWM2 phase delay

The code converts nanoseconds to clock counts using:

```c
#define NS_TO_COUNTS(ns) \
    ((Uint16)((((unsigned long long)(ns)) * ((unsigned long long)DSPCLK) + 500000000ULL) / 1000000000ULL))
```

So for:

```c
#define PWM_DELAY_NS (150U)
```

the phase shift becomes approximately:

```text
150 ns × 150 MHz = 22.5 counts
```

which rounds to about **23 counts**.

## ADC behavior

The ADC is configured to read:

```c
AdcRegs.ADCCHSELSEQ1.bit.CONV00 = 0x0;   // ADCINA0
```

So the monitored analog input is **ADCINA0**.

The ADC is started in software every control-loop cycle:

```c
AdcRegs.ADCTRL2.bit.SOC_SEQ1 = 1;
```

and the code waits until the conversion is complete.

---

## Important note about ADC result formatting

The comment near the threshold says:

```c
// 2.3V threshold on 3.3V full-scale ADC
// 12-bit threshold = 2854
// raw register threshold = 2854 << 4 = 45664
```

But the actual code uses:

```c
#define ADC0_RAW_SHUTDOWN_THRESHOLD   (3186U)
```

and compares it directly against:

```c
return AdcMirror.ADCRESULT0;
```

Later, the code converts the result to voltage as:

```c
gAdcVoltage = ((float)gRawAdcResult * 3.3f) / 4095.0f;
```

That means the code is currently treating the ADC result as a **normal 0–4095 12-bit value**, not as a left-shifted raw register such as 45664.

So the comment is outdated or inconsistent with the actual implementation.

---

## What threshold is currently being used?

The code uses:

```c
#define ADC0_RAW_SHUTDOWN_THRESHOLD   (3186U)
```

With a 3.3 V reference and 12-bit scaling:

```text
Threshold voltage ≈ 3186 × 3.3 / 4095
                 ≈ 2.57 V
```

So the **actual active shutdown threshold is about 2.57 V**, not 2.3 V.

If the intended trip point is 2.3 V, then the threshold should be closer to:

```text
2.3 / 3.3 × 4095 ≈ 2854
```

## Control-loop operation

The control loop is driven by **CPU Timer0 interrupt**.

### Timer configuration

```c
ConfigCpuTimer(&CpuTimer0, 150, (1000000UL / FS_MOD));
```

Since `FS_MOD = 5000`, the timer period becomes:

```text
1,000,000 / 5000 = 200 microseconds
```

So the software loop runs at:

- **5 kHz**
- every **200 µs**

The ISR is very small. It only sets a flag:

```c
TimerFlag = 1;
```

The main loop does the actual work when that flag is set.

This is a good design because it keeps the interrupt short and leaves most processing in the background loop.

---

## Startup ignore logic

At startup, the code ignores ADC trip detection for a number of cycles:

```c
if(gStartupCounter < ADC_STARTUP_IGNORE_COUNT)
{
    gStartupCounter++;
    gTripCondition = 0U;
}
```

With:

```c
#define ADC_STARTUP_IGNORE_COUNT (1000U)
```

and a loop rate of **5 kHz**, this gives:

```text
1000 / 5000 = 0.2 s
```

So ADC trip detection is ignored for about **200 ms after startup**.

This is useful when the analog input needs time to settle.

---

## Fault shutdown logic

After startup ignore ends, the code checks:

```c
gTripCondition = ((gPwmFaultActive == 0U) &&
                  (gRawAdcResult < gRawAdcThreshold)) ? 1U : 0U;
```

If true, it calls:

```c
PWM_Shutdown();
```

That function forces one-shot trip on both PWM modules:

```c
EPwm1Regs.TZFRC.bit.OST = 1;
EPwm2Regs.TZFRC.bit.OST = 1;
```

Because the trip-zone action is configured as:

```c
p->TZCTL.bit.TZA = TZ_FORCE_LO;
p->TZCTL.bit.TZB = TZ_FORCE_LO;
```

both outputs are forced LOW.

---

## Restart logic

Once faulted, the code waits for a software delay:

```c
if(gRestartCounter < PWM_RESTART_DELAY_COUNT)
{
    gRestartCounter++;
}
```

When the delay expires, ADC is read again.

PWM is only restarted if:

```c
if(gRawAdcResult >= gRawAdcThreshold)
{
    PWM_Restart();
    ...
}
```

This means the restart is **conditional**, not automatic.

If the ADC is still below threshold, the PWM remains shut down and the restart timer resets.

---

## How long is the restart delay?

The code defines:

```c
#define PWM_RESTART_DELAY_COUNT  (FS_MOD * 10UL)
```

Since `FS_MOD = 5000`, the restart delay is:

```text
5000 × 10 = 50000 control cycles
```

At 5 kHz:

```text
50000 / 5000 = 10 seconds
```

So the code waits **10 seconds** before attempting restart.

---

## PWM configuration details

Both PWM modules use a shared setup function:

```c
common_epwm_complementary_setup(...)
```

That function configures:

- up-down count mode
- shadow compare loading
- action-qualifier behavior
- deadband generation
- trip-zone low forcing

### Action qualifier behavior

```c
p->AQCTLA.bit.CAU = AQ_CLEAR;
p->AQCTLA.bit.CAD = AQ_SET;
```

This means PWM A is:

- cleared on compare-up
- set on compare-down

which creates a symmetric PWM waveform.

### Deadband

```c
p->DBCTL.bit.OUT_MODE = DB_FULL_ENABLE;
p->DBRED = EPWM_DB;
p->DBFED = EPWM_DB;
```

This enables complementary PWM outputs with both rising-edge and falling-edge deadband.

## Difference between ePWM1 and ePWM2

### ePWM1
```c
EPwm1Regs.TBCTL.bit.PHSEN    = TB_DISABLE;
EPwm1Regs.TBCTL.bit.SYNCOSEL = TB_CTR_ZERO;
```

- phase loading disabled
- acts as sync master

### ePWM2
```c
EPwm2Regs.TBCTL.bit.PHSEN    = TB_ENABLE;
EPwm2Regs.TBCTL.bit.SYNCOSEL = TB_SYNC_IN;
EPwm2Regs.TBPHS.half.TBPHS   = PWM2_DELAY_COUNTS;
```

- phase loading enabled
- syncs from ePWM1
- delayed by `PWM2_DELAY_COUNTS`

So ePWM2 is the phase-shifted follower.

## File flow summary

### `main()`
Initializes everything, then runs the background loop.

### `cpu_timer0_isr()`
Sets `TimerFlag = 1` every timer period.

### `InitAdcA0_F28335()`
Configures ADC sequencer to sample ADCINA0.

### `ReadAdcRawA0_F28335()`
Starts one ADC conversion and returns the result.

### `SetupEPWMs()`
Initializes GPIO and configures both ePWM modules.

### `ConfigureEPwm1()`
Sets ePWM1 as the reference PWM.

### `ConfigureEPwm2()`
Sets ePWM2 as a phase-shifted PWM.

### `PWM_Shutdown()`
Forces one-shot trip and drives PWM outputs low.

### `PWM_Restart()`
Clears one-shot trip so PWM operation can resume.

## Expected real behavior with current settings

With the present configuration, the system behaves like this:

- PWM switching frequency: **70 kHz**
- software monitoring frequency: **5 kHz**
- startup ADC ignore time: **200 ms**
- restart wait time after fault: **10 s**
- phase shift of ePWM2: about **150 ns**
- fault threshold in current code: about **2.57 V**

If ADCINA0 falls below about **2.57 V**, both PWM modules shut down.
After 10 seconds, the system checks the ADC again.
If ADCINA0 has recovered to at least the threshold, PWM resumes.
Otherwise, it stays off and keeps retrying.

## Important inconsistencies / things new users should know

### 1. Threshold comment does not match the active code
The comment says 2.3 V, but `3186` corresponds to about **2.57 V**.

### 2. ADC formatting comment is misleading
The comment talks about left-justified raw values, but the code uses the ADC result as if it is a standard **0–4095** value.

### 3. `gRawAdcResult` is assigned twice
Inside `ReadAdcRawA0_F28335()`:

```c
gRawAdcResult = AdcMirror.ADCRESULT0;
return AdcMirror.ADCRESULT0;
```

and then the caller also assigns:

```c
gRawAdcResult = ReadAdcRawA0_F28335();
```

This is not harmful, but redundant.

### 4. PWM duty is overwritten in the main loop
When no fault is active:

```c
EPwm1Regs.CMPA.half.CMPA = PERIOD / 2;
EPwm2Regs.CMPA.half.CMPA = PERIOD / 2;
```

So duty is forced to 50% every timer cycle.

If a user later wants variable duty cycle, this line must be changed.

## How to change the shutdown threshold

To use a different voltage threshold:

```text
ADC count = Vthreshold / 3.3 × 4095
```

### Example: 2.3 V threshold

```text
2.3 / 3.3 × 4095 ≈ 2854
```

So you would use:

```c
#define ADC0_RAW_SHUTDOWN_THRESHOLD   (2854U)
```

---

## How to change restart delay

The current restart delay is:

```c
#define PWM_RESTART_DELAY_COUNT  (FS_MOD * 10UL)
```

If you want 5 seconds instead of 10 seconds:

```c
#define PWM_RESTART_DELAY_COUNT  (FS_MOD * 5UL)
```

---

## How to change startup ignore time

Current startup ignore:

```c
#define ADC_STARTUP_IGNORE_COUNT (1000U)
```

At 5 kHz, that equals 200 ms.

For 100 ms instead:

```text
0.1 s × 5000 = 500
```

So:

```c
#define ADC_STARTUP_IGNORE_COUNT (500U)
```

## Summary

This code creates a protection-oriented PWM control example on the F28335:

- **ePWM1** and **ePWM2** generate complementary PWM
- **ePWM2** is phase shifted relative to ePWM1
- **ADCINA0** is continuously monitored in a 5 kHz loop
- if the ADC falls below threshold, both PWMs are shut down
- after a 10-second delay, the code checks the ADC again
- PWM restarts only when the analog condition is safe again

This makes it a good lab test framework for ADC-based PWM fault protection and restart behavior.
